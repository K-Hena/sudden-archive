---

# 에러 처리

Supabase 호출(`sb.from(...)`, `sb.auth...`, `sb.storage...`)은
반환되는 `{ data, error }`의 `error` 필드로 확인한다. (Supabase 클라이언트는 실패 시 예외를 던지지 않고 error를 반환한다)

라이브러리 초기화 등 예외가 실제로 발생할 수 있는 코드만 try-catch로 감싼다.

없는 곳에 불필요한 try-catch를 추가하지 않는다.

# AI 규칙

AI는

- 프로젝트 전체 구조를 먼저 이해한 후 작업한다.
- 기존 코드를 존중한다.
- 기존 기능을 임의로 변경하지 않는다.
- 추측하지 않는다.
- 과도한 리팩터링을 하지 않는다.
- 승인 없는 구조 변경을 하지 않는다.
- DB 구조는 임의로 변경하지 않는다.
