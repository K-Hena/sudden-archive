### 그룹 D-2: 대시보드 전면 개편
- D-2a. 대시보드 셸 + 사이드바 메뉴 구조 (편집모드를 감싸는 형태), 통계 탭 (그룹 D-1 클릭수 데이터 활용)
- **D-2b. 항목 생명주기 관리** — 소프트 삭제(휴지통)와 게시 유예를 하나의 상태 관리 설계로 묶어서 진행. 둘 다 `items.status`(예: `pending`/`published`/`trashed`) 같은 공통 필드에 걸리므로, 대시보드 UI 이전에 **전체 조회 쿼리(`renderCards()`, `renderMapGrid()`, 전체 검색 등)와 RLS에 미치는 영향을 먼저 설계해야 하는 선행 작업**입니다.
  - 소프트 삭제: 즉시 삭제 대신 숨김 처리 후 관리자가 검토
  - 게시 유예: 신규 항목 등록 시 자동 N분 유예 + 관리자가 수동으로 즉시 게시하는 혼합형 방향을 **우선 검토** 중 (아직 실 구현·리뷰를 거치지 않은 방향 합의 단계, 확정 아님 — 실제 확정 시 `DECISIONS.md`에 별도 기록)
- D-2c. 댓글 모아보기 탭 (댓글 기능 완료 후)
- D-2d. 문의함 탭 (문의 기능 완료 후)
